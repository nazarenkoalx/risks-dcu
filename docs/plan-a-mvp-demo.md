# ПЛАН А — MVP-демо для инвесторов сделано
## React + Mock Data (без бэкенда, без БД)

---

## Цель

Показать инвесторам рабочий прототип продукта: полный UX-флоу от создания риска до коллегиального голосования и утверждения скора. Никакого реального бэкенда — всё работает на mock-данных в памяти браузера.

**Что должен увидеть инвестор:**
1. Проблему — "риски оцениваются одним человеком, субъективно"
2. Решение — "у нас голосует вся команда"
3. Качество UX — "это не скучная enterprise-форма, это приятный продукт"
4. Бизнес-логику — понятный процесс от идентификации до мониторинга

---

## Стек

```
React 18 + TypeScript + Vite
TailwindCSS + shadcn/ui (компоненты)
React Router v6 (навигация)
Zustand (стейт-менеджмент с mock-данными)
Recharts (тренд-графики)
Custom SVG (Heat Map — не нужна библиотека для 5×5)
Lucide React (иконки)
```

**Деплой**: Vercel / Netlify (статика, бесплатно, URL для презентации)

### Голосования

**Завершённая сессия** (риск r1 — показываем готовые результаты С dispersion):
```typescript
export const completedSession = {
  id: 'vs1',
  riskId: 'r1',
  status: 'completed',
  participants: [
    { userId: 'u2', weight: 3, name: 'Марина Ковалёва' },
    { userId: 'u3', weight: 2, name: 'Дмитрий Орлов' },
  ],
  votes: [
    { userId: 'u2', likelihood: 5, impact: 5, velocity: 5, rationale: 'Видели аналогичный инцидент у конкурентов в прошлом квартале. Угроза реальная и близкая.' },
    { userId: 'u3', likelihood: 2, impact: 4, velocity: 3, rationale: 'Считаю угрозу преувеличенной. Наши меры достаточны.' },
  ],
  consensusLikelihood: 3.86,  // взвешенное среднее (5×3 + 2×2) / 5
  consensusImpact: 4.6,
  consensusScore: 18,         // CRITICAL
  dispersionFlag: true,       // ← stdev > 1.5 по likelihood (5 vs 2) — КЛЮЧЕВОЙ МОМЕНТ ДЕМО
  dispersionMessage: 'Значительное расхождение мнений по вероятности (5 vs 2). Рекомендуется обсуждение перед утверждением.',
  approvedBy: 'u1',
}
```

> **Почему dispersion = true:** инвестор должен увидеть, что CEO поставил 5, а CFO поставил 2.
> Это и есть ценность продукта — без системы этот разговор никогда бы не случился.

**Активная сессия** (риск r2 — запускаем live в демо):

## Экраны (страницы)

### 1. Login Page — split-screen layout

**Левая панель (тёмная, Pantone 4154 C) — Problem Frame:**
```
СЕГОДНЯ

Риск "Утечка данных клиентов"
оценил один аналитик.
Скор: 12 / Высокий.

Совет директоров принял решение
на основе мнения одного человека.
```
→ Анимированная смена на:
```
С RISKBOARD

Три топ-менеджера проголосовали
независимо. CEO: 5, CFO: 2.
Система выявила расхождение.

Состоялся разговор, который
изменил решение.
```

> Инвестор читает проблему и решение за 10 секунд — до того как видит интерфейс.

**Правая панель — вход в демо:**
- Заголовок: "RiskBoard" + слоган
- Две кнопки: "Войти как CRO (Алексей Громов)", "Войти как CEO (Марина Ковалёва)"
- Третья кнопка: "Войти как CFO (Дмитрий Орлов)"
- Быстрый вход без пароля — для демо

### 2. Dashboard (главная)
Три секции:

**KPI-карточки (верхний ряд):**
- Всего рисков: 7
- Критических (score ≥ 15): 2 🔴
- Ожидают голосования: 1 🟡
- С расхождением мнений: 1 ⚠️ ← новая карточка, сразу показывает ценность

**Heat Map 5×5** (центральный элемент):
- Интерактивная SVG-матрица
- Каждая ячейка показывает количество рисков
- Клик по ячейке — фильтр реестра
- Цвета: зелёный (1-4), жёлтый (5-9), оранжевый (10-14), красный (15-25)

**Топ-5 рисков по скору** (правая колонка):
- Список с badges категории, скором, именем владельца

### 3. Risk Register
- Таблица/список всех рисков
- Фильтры: категория, статус, скор
- Колонки: Название, Категория, Статус, Скор (цветной бейдж), Владелец, Следующий пересмотр
- Кнопка "Создать риск"
- Статус-бейджи: Draft / Voting / Assessed / In Treatment / Monitoring

### 4. Risk Detail Page
Полная карточка риска. Две вкладки (убрана Audit Trail — не нужна в демо):

**Обзор**: описание, категория, владелец, даты

**Оценка**: секция голосования
- Если статус `draft` → кнопка "Запустить голосование" (только для coordinator)
  → открывает **VotingSetupModal** (не переход на отдельную страницу):
  ```
  Участники голосования:
  [x] Марина Ковалёва (CEO)    Вес: [3]  → 60% голоса
  [x] Дмитрий Орлов  (CFO)    Вес: [2]  → 40% голоса
  [Запустить голосование]
  ```
- Если статус `voting_in_progress` → прогресс-бар + список участников (✓ / ожидание)
  - Кнопка coordinator: **"Завершить и рассчитать"** (активна при любом числе голосов ≥ 1)
  > Fast-forward: координатор не ждёт 100% явки в live-демо. Это не баг — это фича emergency closure.
- Если статус `assessed` / `monitoring` → результаты: distribution chart + DispersionAlert + consensus score

**Сценарии реагирования**: список pre-filled сценариев с задачами (не редактируем в демо)

### 5. New Risk Page (2 шага — убран шаг со сценариями)

**Шаг 1 — Описание:**
- Название*
- Описание (что может произойти)
- Категория (dropdown)
- Причины возникновения
- Возможные последствия

> Шаг 2 со сценариями удалён из мастера: для инвестора это dead time.
> Сценарии pre-filled для нового риска автоматически (mock). Редактирование — на Risk Detail.

**Шаг 2 — Запуск голосования:**
- Выбор участников из списка (checkbox)
- Установка весов (number input, 1-5)
- Предпросмотр: "Каждый участник будет иметь X% голоса"
- Кнопка "Создать риск и запустить голосование"

> После нажатия: риск создаётся со статусом `voting_in_progress`, переход на Risk Detail.

### ~~6. Voting Setup Page~~ — удалена, заменена VotingSetupModal
Отдельная страница убрана. Настройка голосования — модалка на Risk Detail (см. выше).
Это убирает один лишний переход в демо-флоу и упрощает код.

### 6. Voting Page (КЛЮЧЕВОЙ ЭКРАН ДЛЯ ИНВЕСТОРОВ)
Чистый, минималистичный экран — как будто открыл письмо на телефоне.

**Хедер**: логотип RiskBoard + имя и роль текущего пользователя + **NotificationBadge** "1 голос ожидает" (пульсирующий, Pantone 186 C)

**Заголовок**: "Оцените риск: [название]"
**Краткое описание риска**

**Три слайдера/кнопки-выборщики:**

```
Вероятность наступления
[1] [2] [3] [4] [5]
 Маловероятно      Почти наверняка

Влияние на компанию
[1] [2] [3] [4] [5]
 Незначительное    Критическое

Скорость наступления
[1] [2] [3] [4] [5]
 Медленно          Мгновенно
```

**Поле**: "Ваш комментарий (необязательно)" — textarea

**Кнопка**: "Отправить оценку" (Pantone 2145 C — royal blue)

**После отправки**: "Ваша оценка учтена. Результаты появятся после завершения голосования."
→ Автоматический переход на Dashboard через 2 секунды.

> UI-принцип: топ-менеджер голосует за 2 минуты, не открывая инструкцию.

### 7. Voting Results Page
После завершения сессии — страница с результатами:

**Consensus Score**: большой цветной бейдж "18 / КРИТИЧЕСКИЙ" (Pantone 186 C)

**DispersionAlert** (если dispersion_flag = true) — КЛЮЧЕВОЙ БЛОК:
```
┌─────────────────────────────────────────────────────────┐
│ ⚠  Значительное расхождение мнений                      │
│                                                         │
│ CEO оценил вероятность как 5 (Почти наверняка)          │
│ CFO оценил вероятность как 2 (Маловероятно)             │
│                                                         │
│ Рекомендуется обсуждение перед утверждением финального  │
│ скора. Разные перспективы могут указывать на неполную   │
│ информацию у одного из участников.                      │
└─────────────────────────────────────────────────────────┘
```
> Этот блок — "aha moment" для инвестора. Именно здесь показывается почему voting > one analyst.

**Distribution Chart** (горизонтальные бар-чарты):
```
Вероятность:
  Ковалёва CEO (вес 3): █████ 5  ← красный
  Орлов    CFO (вес 2): ██░░░ 2  ← зелёный
                                    ↑ наглядное расхождение
  Взвешенное среднее: 3.86
```

**Итоговый скор**: Вероятность 3.86 × Влияние 4.6 = **18** (КРИТИЧЕСКИЙ)

**Комментарии участников** (раскрывающиеся блоки с rationale)

**Кнопка Coordinator**: "Утвердить скор" → риск переходит в статус `monitoring`

## Demo Flow (сценарий презентации инвесторам)

**Сценарий: 5 минут, 3 роли, линейный (без возвратов назад)**

```
── ЧАСТЬ 1: ПРОБЛЕМА (30 сек) ──────────────────────────────────────

[Открываем LoginPage]
  Левая панель: "CEO принял решение на основе мнения одного аналитика."
  → Даём зрителям прочитать. Не спешим.

── ЧАСТЬ 2: РЕШЕНИЕ — НАСТРОЙКА (1 мин) ────────────────────────────

[CRO] Входим как Алексей Громов (CRO)
  → Dashboard: Heat Map, 2 критических риска (красные ячейки)
  → Карточка "С расхождением мнений: 1" — акцентируем внимание
  → Кликаем красную ячейку → реестр фильтруется

[CRO] Открываем риск "Изменение требований ЦБ" (статус: Draft)
  → Нажимаем "Запустить голосование"
  → Модалка: выбираем CEO и CFO, выставляем веса (60% / 40%)
  → "Запустить" → статус меняется на Voting in Progress

── ЧАСТЬ 3: ГОЛОСОВАНИЕ (2 мин) ────────────────────────────────────

[Переключаемся: CEO — Марина Ковалёва]
  → В хедере: NotificationBadge "1 голос ожидает" (красный)
  → Кликаем → Voting Page
  → Выставляем оценки (Вероятность: 5, Влияние: 5, Скорость: 5)
  → Комментарий: "Видели аналогичный инцидент у конкурентов"
  → "Отправить оценку" → прогресс-бар 1/2

[Переключаемся: CFO — Дмитрий Орлов]
  → NotificationBadge "1 голос ожидает"
  → Voting Page → Вероятность: 2, Влияние: 4, Скорость: 3
  → Комментарий: "Считаю угрозу преувеличенной"
  → "Отправить" → прогресс-бар 2/2 ✓

── ЧАСТЬ 4: РЕЗУЛЬТАТ — "AHA MOMENT" (1 мин) ───────────────────────

[CRO] Кнопка "Завершить и рассчитать"
  → Страница результатов открывается

  DispersionAlert (красный блок):
  "⚠ CEO оценил вероятность как 5, CFO — как 2.
   Рекомендуется обсуждение."

  → Показываем Distribution Chart: наглядное расхождение
  → Consensus Score: 18 / КРИТИЧЕСКИЙ

[CRO] "Утвердить скор"
  → Risk появляется в Heat Map в ячейке (4, 5) — красной

── ЧАСТЬ 5: ИТОГ (30 сек) ──────────────────────────────────────────

[CRO] Возврат на Dashboard
  → Heat Map обновилась
  → Упоминаем: "В сайдбаре — Action Plan с задачами по этому риску"
  → Не переходим — достаточно сказать
```

> **Правило ведущего**: не отклоняться от этого сценария. Любой "а давайте посмотрим ещё вот это"
> на живой демо — риск потерять нить. Если инвесторы спрашивают — отвечать словами, не кликами.

## Ключевые дизайн-решения

**Цвет скора риска:**
- 1-4: `bg-[#E8F4FA] text-[#003B5C]` — Низкий (sky / navy)
- 5-9: `bg-[#ECEEF8] text-[#5557A0]` — Средний (periwinkle)
- 10-14: `bg-orange-50 text-orange-700` — Высокий (warning orange)
- 15-25: `bg-[#FDECEA] text-[#C8102E]` — Критический (Pantone 186 C)

**Heat Map ячейки (SVG):**
```tsx
const CELL_COLORS = {
  low:      '#9ACEEB', // Pantone 291 C — голубой
  medium:   '#C5C9E4', // Pantone 7451 C light — лавандовый
  high:     '#FFA726', // оранжевый предупреждение
  critical: '#C8102E', // Pantone 186 C — красный
}
// score = likelihood * impact
// low: ≤4, medium: 5-9, high: 10-14, critical: ≥15
```
